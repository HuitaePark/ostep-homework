1. 우선 몇 주소를 변환하기 위해 작은 주소 공간을 사용하여 보자. 여기에 여러 가지
랜덤 시드와 함께 간단한 매개변수의 집합이 있다. 이때 주소를 변환할 수 있을까?  

-> 어드레스 스페이스 사이즈에 따라 
    1. 128의 절반인 64를 기준으로:
        0 ~ 63: Segment 0 (양수 방향으로 자람)
        64 ~ 127: Segment 1 (음수 방향으로 자람)
    2. 역방향인지 정방향인지에 따라 오프셋을 구함
    3. 20으로 범위를 지정했는데 초과했는지 검사
    4. 여기에 물리주소의 맨 끝인 512에서 20거리만큼 빼서 물리주소를 구함

2. 둘다 128을 절반으로 나눈 오른쪽에 위치하므로 음수방향으로 자람  
그러므로 108을 가진 0은 최대값이 127에 딱맞게 됨
반대로 1은 97부터 117까지라 97이 최소

3. --b0 0 --l0 2 --b1 127 --l1 2 
4. 시뮬레이터는 전체 가상 주소 공간(-a 크기) 안에서 균등하게 무작위 주소를 뽑는다. 따라서 90%가 유효하려면"실제로 할당된 세그먼트 크기의 합이 전체 주소 공간의 90%" 가 되도록 만들어야 한다.
5.  세그먼트의 크기를 모두 0으로 만들어 버리면, 어떤 가상 주소가 들어오더라도 허용된 범위를 벗어나게 된다.
